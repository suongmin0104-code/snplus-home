# SN 업무포털 운영 가이드

## 목적

`https://snplus.ai.kr/admin`은 주식회사 에스앤의 내부 업무 확인 창구입니다.

- 공개 홈페이지에는 관리자 링크를 표시하지 않습니다.
- 관리자 로그인 후에만 포털 화면과 연동 상태를 확인할 수 있습니다.
- 실제 견적, 세무, 거래처 자료는 외부 ERP가 연결되기 전까지 저장하거나 표시하지 않습니다.
- 일정과 현장 업무일지는 관리자 전용 Vercel Private Blob 저장소에 보관합니다.

## 1차 제공 범위

- 총책임자와 직원별 전화번호 아이디 로그인
- 직원 최초 등록코드 및 본인 비밀번호 생성
- 직원별 업무 권한과 퇴사자 사용 중지
- 8시간 서명 세션
- 통합 현황 화면
- 견적 ERP 연결 상태
- 세무·회계 ERP 연결 상태
- 거래처와 사내자료 확장 화면
- 월간 일정·현장 업무일지와 현장사진
- 관리자 전용 업무 서식 다운로드
- 견적서·거래명세서 웹 바로 작성과 PDF 출력
- 외부 ERP HTTPS 주소 연결

현재 단계는 업무포털의 안전한 진입점과 통합 화면을 구축한 것입니다. ERP 내부 데이터를 읽고 쓰는 API 연동은 사용 중인 서비스와 권한 정책이 확정된 뒤 진행합니다.

## 일정·현장 업무일지

관리자 메뉴의 `일정·할 일`에서 PC와 휴대폰이 함께 사용하는 월간 업무일지를 엽니다.

- 월간 달력과 선택한 날짜의 일정 목록
- 예정, 진행 중, 완료 상태
- 현장·장소, 시작·종료 시간, 업무 구분, 참여자, 작업내용
- 현장사진 최대 6장
- 휴대폰 카메라 후면 촬영과 사진 선택
- 통합 현황의 미완료 업무와 오늘 일정 건수
- `.ics` 파일을 이용한 휴대폰 기본 캘린더 추가

업무일지는 Vercel Private Blob의 `admin-worklog/entries/` 경로에 일정별 JSON으로 저장합니다. 사진은 `admin-worklog/photos/` 경로에 분리하며 `/api/admin/worklog-photo`가 관리자 세션을 확인한 뒤에만 보여줍니다. 비공개 Blob 원본 URL은 화면에 전달하지 않습니다.

휴대폰 사진은 브라우저에서 긴 변을 최대 1600px로 줄이고 JPEG로 변환한 뒤 한 장당 약 1.8MB 이하로 업로드합니다. 서버에서는 JPEG, PNG, WEBP만 허용하고 한 장당 2MB를 초과하면 거부합니다.

### 휴대폰 사용

1. 휴대폰 브라우저에서 `https://snplus.ai.kr/admin`을 엽니다.
2. PC와 같은 관리자 계정으로 로그인합니다.
3. `일정·할 일`에서 날짜를 선택하고 `새 일정`을 누릅니다.
4. `촬영·사진 선택`으로 현장을 촬영한 뒤 저장합니다.
5. PC에서 같은 관리자 화면을 열면 저장한 기록과 사진이 함께 표시됩니다.

`캘린더 추가`는 Apple 캘린더, Google Calendar 앱 등이 읽을 수 있는 일정 파일을 내려받는 기능입니다. 현재는 한 번 가져오는 방식이며 외부 캘린더에서 수정한 내용이 업무포털로 자동 반영되는 양방향 동기화는 아닙니다. 양방향 자동 동기화에는 Google 또는 Microsoft 계정별 OAuth 권한 설계가 추가로 필요합니다.

## 업무 서식

관리자 메뉴의 `업무 서식`에서 다음 원본 엑셀 파일을 내려받을 수 있습니다.

- 견적서 양식: `z0.견적서양식.xls`
- 거래명세서 양식: `0.거래명세서 양식.xls`

원본은 GitHub 저장소에 커밋하지 않고 Vercel Private Blob의 `admin-templates/` 경로에 보관합니다. `/api/admin/template`이 로그인 세션을 확인한 뒤 고정된 파일 식별자에 해당하는 서식만 Private Blob에서 읽어 전송합니다. 파일 경로를 요청값으로 받지 않기 때문에 임의 경로 접근은 허용되지 않습니다.

Vercel 연결값:

```text
BLOB_READ_WRITE_TOKEN=Vercel Storage 연결 시 자동 등록
```

운영 함수에서는 Vercel OIDC 또는 연결된 Blob 토큰을 사용합니다. 토큰과 비공개 Blob URL은 코드, 문서, GitHub에 기록하지 않습니다.

## 웹 문서 작성

관리자 메뉴의 `문서 작성` 또는 `업무 서식`의 `바로 작성`에서 견적서와 거래명세서를 엽니다.

- 거래처, 날짜, 공사명, 품목, 규격, 단위, 수량, 단가, 비고 입력
- 수량과 단가를 이용한 품목별 금액 자동 계산
- 부가세 별도, 포함, 면세 선택
- 공급가액, 부가세, 합계금액과 한글 금액 자동 표시
- 품목 행 추가·삭제
- 문서 종류별 현재 브라우저 자동 임시저장
- `PDF 저장 / 인쇄` 버튼으로 A4 인쇄 화면 실행

브라우저 인쇄 화면의 대상에서 `PDF로 저장`을 선택하면 PDF 파일로 저장할 수 있습니다. 작성 내용은 서버나 GitHub에 저장하지 않고 해당 브라우저의 로컬 저장소에만 임시 보관합니다. 공용 PC에서는 작성 후 `새 문서`로 초기화하거나 브라우저 사이트 데이터를 삭제합니다.

## 총책임자 계정 준비

Vercel 프로젝트가 연결된 PC에서는 다음 명령으로 계정과 보안값을 한 번에 등록할 수 있습니다.

```powershell
npm run admin:configure -- 관리자아이디
```

비밀번호는 입력 개수만 별표로 표시되며 원문이나 해시를 파일에 남기지 않고 Vercel Production 환경변수로 바로 전송합니다.

수동으로 환경변수를 등록해야 할 때는 아래 해시 생성 명령을 사용합니다.

저장소 폴더의 터미널에서 다음 명령을 실행합니다.

```powershell
npm run admin:hash
```

화면에 비밀번호가 표시되지 않는 상태로 비밀번호를 두 번 입력하면 다음 두 값이 생성됩니다.

```text
ADMIN_PASSWORD_HASH=...
ADMIN_SESSION_SECRET=...
```

이 값은 코드나 문서에 저장하지 말고 Vercel 환경변수에만 등록합니다.

## Vercel 필수 환경변수

```text
ADMIN_USERNAME=
ADMIN_PASSWORD_HASH=
ADMIN_SESSION_SECRET=
```

- `ADMIN_USERNAME`: 관리자 로그인 아이디
- `ADMIN_PASSWORD_HASH`: `npm run admin:hash`로 만든 해시
- `ADMIN_SESSION_SECRET`: 같은 명령으로 만든 임의 세션 키
- 세 값 중 하나라도 없으면 관리자 페이지는 열리지 않습니다.

환경변수 등록 후 Production 재배포가 필요합니다.

## 직원 계정과 권한 운영

총책임자 로그인 후 `직원·권한` 메뉴에서 직원 계정을 관리합니다.

1. `직원 등록`을 누릅니다.
2. 이름, 직급, 휴대전화 번호와 사용할 업무 권한을 선택합니다.
3. 화면에 한 번 표시되는 전화번호 아이디와 최초 등록코드를 직원에게 전달합니다.
4. 직원은 `/admin`의 `첫 비밀번호 만들기`에서 전화번호, 등록코드, 새 비밀번호를 입력합니다.
5. 등록이 끝난 뒤부터 전화번호와 본인이 만든 비밀번호로 로그인합니다.

비밀번호는 10자 이상이며 문자와 숫자를 함께 사용합니다. 총책임자는 직원 비밀번호를 보거나 대신 정하지 않습니다. 비밀번호를 잊은 경우 `비밀번호 재등록`으로 기존 비밀번호를 무효화하고 새 등록코드를 발급합니다.

현재 권한 구분:

- 견적 관리·문서 작성
- 세무·회계
- 거래처
- 일정·현장 업무일지
- 업무 서식·사내 자료

퇴사·휴직자는 `사용 중지`로 처리합니다. 계정 상태는 각 API 요청에서 다시 확인하므로 이미 로그인된 브라우저도 다음 요청부터 차단됩니다. 총책임자 기본 계정은 Vercel 환경변수에 유지하며 화면에서 삭제하거나 권한을 낮출 수 없습니다.

직원 계정은 Vercel Private Blob의 `admin-users/entries/`에 전화번호별 JSON으로 보관합니다. 비밀번호와 최초 등록코드는 `scrypt` 해시만 저장하고 원문은 저장하지 않습니다.

## ERP 바로가기 환경변수

사용 중인 ERP의 공식 HTTPS 주소가 정해지면 다음 값을 등록합니다.

```text
ESTIMATE_ERP_URL=https://...
TAX_ERP_URL=https://...
```

이 단계에서는 ERP 로그인 페이지를 새 창으로 연결합니다. ERP 아이디와 비밀번호는 SN 홈페이지에서 받거나 저장하지 않습니다.

## 실제 데이터 통합 전 확인사항

견적 ERP:

- 사용 서비스 또는 자체 프로그램 이름
- 공식 접속 주소
- API 제공 여부와 API 문서
- 견적, 거래처, 수주 상태 중 표시할 범위
- 사용자별 열람 및 수정 권한

세무·회계 ERP:

- 사용 서비스 또는 세무대리인 시스템 이름
- 공식 접속 주소
- API 제공 여부와 API 문서
- 신고 일정, 매입·매출, 증빙 중 표시할 범위
- 개인정보 및 세무자료 보관 정책

## 보안 원칙

- 관리자 비밀번호 원문을 코드, GitHub, 브라우저 저장소에 저장하지 않습니다.
- 세션 쿠키는 `HttpOnly`, `Secure`, `SameSite=Strict`로 발급합니다.
- 관리자 페이지는 검색엔진에서 제외합니다.
- 외부 ERP 주소는 HTTPS만 허용합니다.
- 세무자료와 민감한 회사자료는 공개 `assets` 폴더에 두지 않습니다.
- 관리자 업무 서식 원본은 Public GitHub 저장소에 커밋하지 않고 Private Blob에만 보관합니다.
- 실제 ERP 데이터를 붙이기 전에는 MFA, 역할별 권한, 감사 로그, 백업 정책을 추가 설계합니다.

## 로컬 화면 확인

관리자 화면 디자인만 확인할 때:

```powershell
npm run dev
```

```text
http://127.0.0.1:5173/admin.html?ui-preview=1
http://127.0.0.1:5173/admin.html?ui-preview=1&module=document-editor&doc=estimate
http://127.0.0.1:5173/admin.html?ui-preview=1&module=tasks
```

`ui-preview=1`은 Vite 개발 모드에서만 동작하며 운영 빌드에서는 관리자 로그인을 우회하지 않습니다.

## 운영 점검

1. `/admin`에서 로그인 화면이 표시되는지 확인합니다.
2. 잘못된 비밀번호가 거부되는지 확인합니다.
3. 로그인 후 통합 현황이 표시되는지 확인합니다.
4. 로그아웃 후 관리자 API가 다시 거부되는지 확인합니다.
5. ERP 주소가 없을 때 `연결 전`으로 표시되는지 확인합니다.
6. ERP 주소가 있을 때 새 창으로만 열리는지 확인합니다.
7. 공개 홈페이지에 관리자 링크가 없는지 확인합니다.
8. 로그인 전 업무 서식 다운로드가 거부되는지 확인합니다.
9. 로그인 후 견적서와 거래명세서가 원본 `.xls`로 내려받아지는지 확인합니다.
10. 웹 문서에서 수량과 단가를 입력했을 때 합계와 부가세가 계산되는지 확인합니다.
11. 견적서와 거래명세서 전환 시 각각의 임시 문서가 유지되는지 확인합니다.
12. `PDF 저장 / 인쇄` 결과가 A4 한 장에 잘리지 않고 표시되는지 확인합니다.
13. PC와 휴대폰에서 같은 업무일지가 표시되는지 확인합니다.
14. 휴대폰 촬영 사진이 저장 후 다시 표시되는지 확인합니다.
15. 로그인하지 않은 상태에서 업무일지와 사진 API가 거부되는지 확인합니다.
16. 일정 삭제 시 연결된 비공개 사진도 함께 삭제되는지 확인합니다.
17. 총책임자가 직원 전화번호와 권한을 등록할 수 있는지 확인합니다.
18. 직원이 최초 등록코드로 본인 비밀번호를 만들 수 있는지 확인합니다.
19. 권한이 없는 메뉴가 숨겨지고 해당 API도 403으로 거부되는지 확인합니다.
20. 직원 사용 중지 후 기존 로그인 세션의 다음 요청이 거부되는지 확인합니다.
