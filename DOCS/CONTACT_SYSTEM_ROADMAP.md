# 문의 시스템 확장 로드맵

## 1차: 이메일 접수

현재 구현 단계입니다.

- 견적센터 폼 운영 필드 구성
- `/api/contact` Vercel Serverless Function
- Resend API 메일 발송
- honeypot 스팸 방지
- 성공/실패 안내 메시지

## 2차: Supabase DB 저장

문의 접수 내용을 데이터베이스에 함께 저장합니다.

- 문의 고유번호
- 접수 시간
- 회사명, 담당자명, 연락처, 이메일
- 문의 유형, 제목, 내용
- 처리 상태
- 관리자 메모

## 3차: 관리자 로그인

관리자만 문의 내용을 확인할 수 있게 로그인 기능을 추가합니다.

- 관리자 계정
- 세션 관리
- 접근 권한 확인
- 비밀번호 재설정 정책

## 4차: 문의 목록

관리 화면에서 접수된 문의를 표 형태로 확인합니다.

- 최신순 목록
- 처리 상태 표시
- 중요 문의 표시
- 페이지네이션

## 5차: 문의 상세보기

각 문의별 상세 내용을 확인합니다.

- 입력 내용 전체 보기
- 접수 페이지와 User Agent 확인
- 관리자 답변 메모
- 첨부파일 기능 검토

## 6차: 처리 상태 관리

문의 처리 흐름을 관리합니다.

- 접수
- 확인 중
- 견적 준비
- 답변 완료
- 보류
- 스팸

## 7차: 문자/SMS 알림

새 문의 접수 시 담당자에게 문자 알림을 보냅니다.

- 알림 대상 번호 관리
- 업무시간 외 알림 정책
- 실패 재시도 정책

## 8차: 카카오 알림톡

카카오 알림톡을 통해 접수 및 처리 상태를 안내합니다.

- 비즈니스 채널 연동
- 알림톡 템플릿 승인
- 문의 접수 알림
- 처리 완료 알림

## 우선순위

1. 이메일 접수 안정화
2. DB 저장
3. 관리자 로그인과 목록
4. 상태 관리
5. SMS 또는 카카오 알림
